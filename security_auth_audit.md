# Authentication Coverage Audit (Static Analysis)

This report flags PHP files that appear to accept external input (`$_GET`, `$_POST`, `$_REQUEST`, `$_COOKIE`) and execute database operations, but do not contain an obvious `do_login(...)` call or explicit early session-auth denial check in the file itself.

## Scope and method
- Scanned `*.php` files in the repo, excluding include/data-heavy paths likely not direct routes: `incs/`, `portal/incs/`, `tables/`, `csv_import_scripts/`.
- Marked a file as **protected** only if it has `do_login(...)` or an explicit session gate pattern near the top.
- Marked as **at-risk** when it has both DB usage and external input usage without those protections.
- This is a conservative static pass; some files may still be indirectly protected by front-controller/web-server routing, but they should be reviewed as potentially directly reachable.

- Files flagged as at-risk (DB + external input + no obvious auth gate): **289**
